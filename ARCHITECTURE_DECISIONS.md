# DMBSearchBuilder Architecture Decisions

## Project-specific section

When copying this file to another PageBuilder ecosystem project, update this section first.

- Project name: `DMBSearchBuilder`
- Architectural role: prebuild crawler and generated search-index writer.
- Main stability concerns: crawl scope, excluded paths, URL normalization, text normalization, keyword extraction, SQLite schema, and database compatibility with `DMBSearchViewer`.
- Documentation target: DocumentationBuilder output rendered in `labs_idemobi_com`.

## ADR-001: Generated database compatibility is part of the public contract

- Context: `DMBSearchViewer` consumes the SQLite database generated by this package.
- Decision: Keep database shape, stored URL format, titles, descriptions, and keyword fields backward compatible unless the user explicitly approves a breaking change.
- Consequence: Schema or record-shape changes must be documented with the matching viewer impact.

## ADR-002: Crawl scope must be deterministic

- Context: Website crawling can easily index duplicate or undesired pages.
- Decision: Keep path filtering, query-string removal, and default exclusions deterministic.
- Consequence: Changes to exclusions, redirects, or URL normalization require README and XML documentation updates.

## ADR-003: DocumentationBuilder-first documentation

- Context: Documentation is expected to be extracted and rendered into `labs_idemobi_com`.
- Decision: Public APIs must include useful XML documentation and stable references.
- Consequence: AI prepares documentation and must not claim DocumentationBuilder execution unless it was actually run.
