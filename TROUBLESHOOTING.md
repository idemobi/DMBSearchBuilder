# DMBSearchBuilder Troubleshooting

## Project-specific section

When copying this file to another PageBuilder ecosystem project, update this section first.

- Project name: `DMBSearchBuilder`
- Main troubleshooting areas: crawl scope, excluded paths, URL normalization, missing titles, empty descriptions, keyword extraction, database paths, SQLite writes, and SearchViewer compatibility.
- Documentation target: `labs_idemobi_com`

## Crawled pages are missing

1. Confirm the page is under the configured `BaseUri`.
2. Check `ExcludedPathPrefixes`.
3. Confirm query-string removal is not collapsing distinct pages unexpectedly.
4. Check maximum page limits.
5. Verify redirects remain inside the intended crawl scope.

## Search results have weak text

1. Confirm titles and descriptions are present on the crawled pages.
2. Review text normalization behavior before changing keyword extraction.
3. Check whether boilerplate content is dominating page text.
4. Keep keyword extraction deterministic unless a new strategy is explicitly requested.

## Database is missing or incompatible

1. Confirm the configured database path points to the expected host output folder.
2. Confirm the database was generated by the expected SearchBuilder version.
3. Check that `DMBSearchViewer` is configured to read the same database path.
4. Do not change schema or record shape without documenting the viewer impact.

## Documentation output quality is weak

1. Confirm XML documentation explains crawler inputs, generated outputs, and compatibility constraints.
2. Avoid copied rendering or component wording from unrelated packages.
3. Keep examples focused on build options, excluded paths, and generated database usage.
